Introduction

Back to the party

Lets start with a simple model using the previous data and then plot it:


tree <- ctree(bad ~ ., data = credit, control = ctree_control(maxdepth = 3))

plot(tree, gp = gpar(fontsize = 8))

Now the ggpartys version:


library(ggparty)

# from the example of ?ggparty
ggparty(tree) +
  geom_edge() +
  geom_edge_label() +
  geom_node_label(aes(label = splitvar), ids = "inner") +
  geom_node_label(aes(label = info), ids = "terminal")

Oh! It looks promising but is kind of far of what ggparty can do.

In summary:

  • The function ggparty::ggparty parse the tree (a party object) to a data.frame so you can use ggplot to show the parsed data.
  • Each row is a node with the related informaction: the id, position (x and y), parent, kids, etc.
  • And the magic keep going with the nodedata_* colums which have the data related in that node!

Thats why if you see the info column, you can notice is full of NAs and then the part label = info is showing us just the same NAs.

So, we can do? A lot! In fact, if you are if you are familiarized with tibbles and some purrr functions you now is easy operate list-columns.


library(scales) # for comma,percent function

ggparty(tree) +
  geom_edge() +
  geom_edge_label() +
  geom_node_label(aes(label = splitvar), ids = "inner") +
  geom_node_label(aes(label = info), ids = "terminal") +
  geom_node_label(
    aes(
      label = str_c(
        "N: ",
        comma(nodesize),
        "\nTarget rate: ",
        percent(map_dbl(nodedata_bad, ~ mean(.x == "bad")))
        )
      ),
  size = 2,
  ids = "terminal")
